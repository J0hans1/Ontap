Changes

Promille kalkulator

We have changed the style of the radio buttons. The buttons in Figma were rounded but the buttons in our actual website are squared with a border radius. We have changed the style because our website contains alot of squares and we thought that squared radio buttons would make our webdesign more consistent.

How To

We have decided to change the name of "how to" page to "Brygge selv" because in this page we have som tutorials about brewing. And "Brygge selv" would represent the page better than "How to" On the "How To" page we decided to have the outher box in in a light-blue color and the inner in a dark-blue color. This was in order to have page look as similar to the other pages as possible. On figma you can see the outher box in a dark-blue color and the inner in light-blue. Here we have a theme that goes across all the pages of the site, thus making the site feel complete and planned.

Additions

Navbar

In the navigationbar we initially had lines under each of the subsections of the site. This is illustrated in Figma. We changed this because we found not to be the most asteticly pleasing design. What page you are on the website is indicated by a light-blue background-color on the current subsection on the navbar. This was made in order to make it easier for the user to understand where he/she is at any moment.

"Phone mode"

When the page becomes very small, the logo becomes a compressed version, where "Ontaps bryggelaug" is removed. this gives a smaller section dedicated to the logo. The navbar becomes a dropdown menu, with an arrow pointing to downwards, to indicate that if activated, the menu will open downwards. When menu is activated. the buttons will show in a box, and the arrow at the bottom points upwards, to indicate that the menu will close upwards.

Kontakt Oss

We decided to add a page to our site called "Contact us". This was in order to have page where the user could easily reach out to ontappd. The user types in his/her name, subject and message. When everything is filled out the user can hit send and is then directed to his/her mail. From here the user can easily edit the message or just send it. In addition we needed the site to have anoter distinct js funtion.

Checklist on How To

We decided to add a checklist on the "How To" page in order to let the user have easy controll over the process of making the beer as he is watching the videos on the page. Another reason for this was in order to have one more distinctive js function.

Additional JS files

We decided to seperate the individual scripts for each side, from the main script that all the pages have in common. we called these unique scripts components and organized them into a folder with the same name. with the same name js file for each of the pages on our site in order to make it easier to navigate and understand which component belonged to what site. We still have a single js file "script.js" whith all the funcions that are present on each of the pages on the site. This was in order to ensuere modularity. This pages includes the topbox, navbar and footer. he new js files that were added consists of the following:

Removals

Phone number:

We have removed the phone number from the footer. Because our client "Ontap" does not have a phone number.

Navbar underlining

In the navigationbar we initially had lines under each of the subsections of the site. This is illustrated in Figma. We changed this because we found it not to be the most asteticly pleasing design.

Slideshow

In the minimum requirements it was specified that the slideshow would be automatic, with the ability to change slide whenever the user wanted, with the use of buttons. We figured that for a better user experience it would be better to abandon the idea of an automatic slideshow. We found that it was frustrating to have the slide automatically change, if we wanted to read the text and observe the image for a longer time. Therefore we ended up with a slideshow that only changes when the user wants it to, by clicking buttons.

Modularity

In order to ensure we fulfilled the requirement for modularity we made a js file called "script.js" which had all the functions that were present on more than one page. This includes the navigationbar which can send the user to all the different pages on our site, the topbox with the Ontappd logo and the two beers, and the footer which has links to Ontappd on different platforms. This was primarily done by Markus and Oskar, but everyone played a part here helping each other.
