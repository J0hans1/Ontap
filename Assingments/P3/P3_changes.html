Our client is an interest group for informatics students at NTNU Trondheim. Ontap is an interest group for students who like brewing beer. When we got an assignment from our lecturer to build a website. Our group sent an email asking Ontap if they wanted us to build a new website for them. They wanted a website, so we sat up a meeting, where they explained what they were requiring from this website.

Changes

How To

On the "Hot To" page we decided to have the outher box in in a light-blue color and the inner in a dark-blue color. This was in order to have page look as similar to the other pages as possible. On figma you can see the outher box in a dark-blue color and the inner in light-blue. Here we have a theme that goes across all the pages of the site, thus making the site feel completed and planned out.

Slideshow

In the minimum requirements it was specified that the slideshow would be automatic, with the ability to change slide whenever the user wanted, with the use of buttons. We figured that for a better user experience it would be better to abandon the idea of an automatic slideshow. We found that it was frustrating to have the slide automatically change, if we wanted to read the text and observe the image. Therefore we ended up with a slideshow that only changes when the user wants it to, by clicking buttons.

Additions

Navbar

In the navigationbar we initially had lines under each of the subsections of the site. This is illustrated in Figma. We changed this because we found not to be the most asteticly pleasing design. What page you are on the website is indicated by a light-blue background-color on the current subsection on the navbar. This was made in order to make it easier for the user to understand where he/she is at any moment.

Kontakt Oss

We desided to add a page to our site called "Contact us". This was in order to have page where the user could easily reach out to ontappd. The user types in his/her name, subject and message. When everything is filled out the user can hit send and is then directed to his/her mail. From here the user can easily edit the message or just send it. In addition we needed the site to have anoter distinct js funtion.

Checklist on How To

We decided to add a checklist on the "How To" page in order to let the user have easy controll over the process of making the beer as he is watching the videos on the page. Another reason for this was in order to have one more distinctive js function.

Additional JS files

We still have a single js file "script.js" whith all the funcions that are present on each of the pages on the site. This was in order to ensuere modularity.
